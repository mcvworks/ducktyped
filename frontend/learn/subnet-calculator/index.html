Subnet Calculator Explained

Understand how IP subnetting works, learn to read CIDR notation and subnet masks, calculate network ranges, and plan efficient address allocation with duckTyped's subnet calculator.

What Is Subnetting?

Subnetting is the practice of dividing a single IP network into two or more smaller, logical networks called subnets. Each subnet functions as an independent network segment with its own range of IP addresses, network address, and broadcast address. Subnetting is a foundational concept in network engineering and is essential for anyone managing routers, firewalls, cloud infrastructure, or even a home lab.

Without subnetting, every device on a network would share the same broadcast domain, leading to excessive broadcast traffic, security concerns, and inefficient use of IP address space. By breaking a large network into smaller subnets, administrators can isolate traffic between departments, improve performance, enforce security policies, and conserve IP addresses.

IP Address Structure

An IPv4 address is a 32-bit number, typically written in dotted decimal notation as four octets separated by periods -- for example, 192.168.1.100. Each octet represents 8 bits, so the full address ranges from 0.0.0.0 to 255.255.255.255.

Every IP address is logically divided into two parts:

  • Network portion -- identifies the specific network (subnet) the device belongs to. All devices on the same subnet share the same network bits.
  • Host portion -- identifies the individual device within that network. Each device on the subnet must have a unique host portion.

The boundary between the network portion and the host portion is determined by the subnet mask. Understanding this boundary is the key to all subnet calculations.

Subnet Masks -- Dotted Decimal and CIDR Notation

A subnet mask is a 32-bit value that indicates which bits of an IP address belong to the network and which belong to the host. In binary, a subnet mask is a contiguous sequence of 1s (network bits) followed by a contiguous sequence of 0s (host bits).

There are two common ways to express a subnet mask:

  • Dotted decimal notation -- the mask is written as four octets, for example 255.255.255.0. This format is used in older configurations and operating system network settings.
  • CIDR (Classless Inter-Domain Routing) notation -- the mask is written as a slash followed by the number of network bits, for example /24. This compact format is standard in modern networking, cloud platforms, and firewall rules.

The mask 255.255.255.0 is equivalent to /24 because the first 24 bits are set to 1. Similarly, 255.255.0.0 equals /16, and 255.0.0.0 equals /8.

Calculating Network, Broadcast, and Host Range

Given an IP address and a subnet mask, you can calculate three critical values:

Network Address

The network address is the first address in a subnet and identifies the subnet itself. It is obtained by performing a bitwise AND between the IP address and the subnet mask. For example, given 192.168.1.100/24:

IP address:    192.168.1.100  = 11000000.10101000.00000001.01100100
Subnet mask:   255.255.255.0  = 11111111.11111111.11111111.00000000
Network addr:  192.168.1.0    = 11000000.10101000.00000001.00000000

The network address cannot be assigned to a host device.

Broadcast Address

The broadcast address is the last address in a subnet. It is obtained by setting all host bits to 1. For 192.168.1.0/24, the broadcast address is 192.168.1.255. Packets sent to the broadcast address are delivered to every device on the subnet. Like the network address, it cannot be assigned to a host.

Usable Host Range

The usable host range consists of all addresses between the network address and the broadcast address. For 192.168.1.0/24, the range is 192.168.1.1 through 192.168.1.254, giving 254 usable addresses. The formula for the number of usable hosts is 2^(32 - prefix) - 2, where the subtraction of 2 accounts for the network and broadcast addresses.

Private IP Address Ranges

RFC 1918 defines three blocks of IP addresses reserved for private use. These addresses are not routable on the public internet and are used within internal networks:

  • 10.0.0.0/8 (10.0.0.0 -- 10.255.255.255) -- 16,777,216 addresses. Commonly used in large enterprise networks and cloud VPCs.
  • 172.16.0.0/12 (172.16.0.0 -- 172.31.255.255) -- 1,048,576 addresses. Often used in corporate networks and container orchestration platforms.
  • 192.168.0.0/16 (192.168.0.0 -- 192.168.255.255) -- 65,536 addresses. The most common range for home and small office networks.

Network Address Translation (NAT) allows devices with private addresses to communicate with the public internet by translating their addresses through a router or firewall.

VLSM -- Variable Length Subnet Masking

VLSM (Variable Length Subnet Masking) allows you to use different subnet mask sizes within the same network, rather than applying a single uniform mask. This enables more efficient address allocation by sizing each subnet according to the actual number of hosts it needs to support.

For example, suppose you have the network 192.168.1.0/24 and need to create subnets for three departments: one with 100 hosts, one with 50 hosts, and one with 10 hosts. Instead of using three /24 networks (wasting hundreds of addresses), you could allocate:

  • 192.168.1.0/25 -- 126 usable hosts for the first department
  • 192.168.1.128/26 -- 62 usable hosts for the second department
  • 192.168.1.192/28 -- 14 usable hosts for the third department

VLSM is essential for efficient IP address management in modern networks and is a prerequisite for routing protocols like OSPF and EIGRP.

Supernetting (Route Aggregation)

Supernetting is the inverse of subnetting -- it combines multiple smaller networks into a single, larger network prefix. This technique is also called route aggregation or route summarization. For example, the four networks 192.168.0.0/24, 192.168.1.0/24, 192.168.2.0/24, and 192.168.3.0/24 can be aggregated into a single 192.168.0.0/22 route.

Supernetting reduces the size of routing tables, decreases memory usage on routers, and speeds up routing lookups. It is a critical technique for internet backbone routers that must handle hundreds of thousands of routes.

Practical Examples

Example 1: Home Network

A typical home router assigns addresses from 192.168.1.0/24. The router itself is usually 192.168.1.1, and devices receive addresses from 192.168.1.2 through 192.168.1.254 via DHCP. This provides 253 usable addresses -- more than enough for most homes.

Example 2: Cloud VPC

When creating a VPC in AWS or Azure, you might define a 10.0.0.0/16 network and then carve it into subnets: 10.0.1.0/24 for public web servers, 10.0.2.0/24 for application servers, and 10.0.3.0/24 for databases. Each subnet can have its own route table and security group rules.

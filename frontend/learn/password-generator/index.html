Password Security Explained

Understand how password entropy determines strength, learn how attackers crack passwords, follow modern best practices from NIST, and generate cryptographically secure passwords with duckTyped's password generator.

Why Passwords Matter

Passwords remain the most widely used authentication method on the internet. Despite the rise of biometrics, hardware keys, and passwordless authentication, the vast majority of online accounts -- email, banking, social media, cloud services -- are protected by passwords. A weak password is the single easiest way for an attacker to gain unauthorized access to your accounts and data.

The scale of the problem is staggering. Billions of username-password pairs have been leaked in data breaches over the past decade. Attackers compile these into massive dictionaries and use automated tools to test stolen credentials against thousands of services simultaneously. If you reuse a password across multiple sites, a breach at one service can compromise all of them.

Password Entropy

Entropy is the mathematical measure of a password's unpredictability, expressed in bits. The higher the entropy, the more guesses an attacker must make to crack the password by brute force. The formula for calculating entropy is:

Entropy (bits) = log2(pool_size ^ length)
               = length * log2(pool_size)

Where pool_size is the number of possible characters in each position and length is the number of characters in the password. For example, a 12-character password using uppercase letters, lowercase letters, digits, and 32 special characters has a pool size of 94 and an entropy of:

12 * log2(94) = 12 * 6.55 = 78.6 bits

Character Pools and Entropy per Character

The entropy contributed by each character depends on the size of the character pool:

As a general guideline, passwords with fewer than 40 bits of entropy are considered weak, 40 to 60 bits are moderate, 60 to 80 bits are strong, and anything above 80 bits is very strong. Modern security recommendations typically aim for at least 70-80 bits of entropy.

How Attackers Crack Passwords

Understanding attack methods is essential for creating passwords that resist them. Here are the most common techniques:

Brute Force Attacks

A brute force attack systematically tries every possible combination of characters until the correct password is found. The time required depends on the password's entropy and the attacker's hardware. Modern GPUs can compute billions of hash attempts per second. A password with 40 bits of entropy (about 1 trillion combinations) can be cracked in seconds on commodity hardware, while a password with 80 bits of entropy would take millions of years.

Dictionary Attacks

Rather than trying every combination, a dictionary attack tests a precompiled list of common passwords, words, phrases, and known patterns. These dictionaries include millions of entries compiled from leaked password databases, common substitutions (like p@ssw0rd), and keyboard patterns (like qwerty123). Dictionary attacks are far more efficient than brute force because they exploit the fact that humans tend to choose predictable passwords.

Credential Stuffing

Credential stuffing uses username-password pairs stolen from one data breach to attempt login at other services. Because many users reuse passwords across sites, attackers can compromise accounts at unrelated services using previously leaked credentials. Automated tools can test millions of credential pairs against a target service in a matter of hours.

Rainbow Tables

A rainbow table is a precomputed lookup table that maps password hashes back to their plaintext equivalents. Instead of hashing each guess in real time, an attacker compares a stolen hash against the table for an instant match. Rainbow tables are effective against unsalted hashes (where the same password always produces the same hash). Modern systems defend against this by using salted hashes -- appending a random value to each password before hashing, making precomputed tables useless.

Common Password Mistakes

These are the patterns attackers check first:

  • Short passwords -- anything under 8 characters can be brute-forced in seconds to minutes.
  • Dictionary words -- single English words, names, and common phrases are in every attack dictionary.
  • Predictable substitutions -- replacing a with @, o with 0, or s with $ does not meaningfully increase security. Attackers include these variations in their dictionaries.
  • Keyboard patterns -- sequences like qwerty, 123456, asdfgh, and zxcvbn are among the most commonly used passwords worldwide.
  • Personal information -- birthdays, pet names, phone numbers, and addresses are easily discovered through social media or public records.
  • Password reuse -- using the same password across multiple services means a single breach compromises all of them.
  • Appending numbers or years -- adding 123 or 2024 to the end of a weak password does not make it strong.

Password Length vs. Complexity

There is a widespread misconception that complexity (mixing uppercase, lowercase, digits, and symbols) matters more than length. In reality, length has a far greater impact on entropy than complexity. Consider these comparisons:

  • An 8-character password using all 94 printable ASCII characters: 8 * 6.55 = 52.4 bits
  • A 16-character password using only lowercase letters: 16 * 4.70 = 75.2 bits

The longer, simpler password is significantly stronger. This is why modern guidelines emphasize length over arbitrary complexity requirements. A long passphrase made of random words is both more secure and easier to remember than a short, complex string of characters.

NIST Guidelines (SP 800-63B)

The National Institute of Standards and Technology (NIST) publishes password guidelines in Special Publication 800-63B that have significantly shifted industry best practices. Key recommendations include:

  • Minimum length of 8 characters, with a recommended minimum of 15 characters for higher security. Systems should support passwords of at least 64 characters.
  • No composition rules -- NIST advises against requiring specific character types (uppercase, lowercase, digit, symbol) because these rules lead to predictable patterns and user frustration without meaningfully improving security.
  • No mandatory password rotation -- forced periodic password changes (every 30 or 90 days) cause users to choose weaker passwords and make minimal, predictable modifications. Passwords should only be changed when there is evidence of compromise.
  • Check against breached password lists -- new passwords should be screened against databases of known compromised passwords and rejected if found.
  • No password hints or knowledge-based questions -- these weaken security by providing attackers with clues.
  • Support paste functionality -- password fields should allow pasting from clipboard to encourage the use of password managers.

Passphrases

A passphrase is a password composed of multiple random words, such as correct horse battery staple. Passphrases leverage length to achieve high entropy while remaining memorable. A 4-word passphrase chosen from a dictionary of 7,776 words (as in the Diceware method) provides approximately 51.7 bits of entropy. A 6-word passphrase from the same dictionary provides about 77.5 bits -- sufficient for most applications.

The key requirement is that the words must be chosen randomly, not selected by the user. Humans are poor at generating random sequences, and phrases that seem random to us (like song lyrics or movie quotes) are easily guessed by attackers who include common phrases in their dictionaries.

Password Managers

A password manager is a tool that generates, stores, and auto-fills unique, high-entropy passwords for each of your accounts. You protect all stored passwords with a single strong master password or passphrase. Password managers solve the fundamental problem of password security: the need to use a unique, complex password for every service without being able to memorize them all.

Reputable password managers include 1Password, Bitwarden, KeePassXC, and Dashlane. Most browsers also include built-in password managers, though dedicated tools typically offer stronger features like breach monitoring and secure sharing.

Two-Factor and Multi-Factor Authentication (2FA / MFA)

Two-factor authentication (2FA) adds a second verification step beyond your password. Even if an attacker obtains your password, they cannot access your account without the second factor. Common second factors include:

  • TOTP (Time-based One-Time Password) -- apps like Google Authenticator, Authy, or 1Password generate 6-digit codes that change every 30 seconds.
  • Hardware security keys -- physical devices like YubiKey or Titan Security Key that use the FIDO2/WebAuthn protocol. These are the most phishing-resistant option.
  • SMS codes -- a one-time code sent via text message. While better than no 2FA, SMS is vulnerable to SIM-swapping attacks and should be considered the weakest option.
  • Push notifications -- authentication apps that send a push notification to your phone for approval.

Enabling 2FA on all critical accounts (email, banking, cloud services, social media) is one of the most impactful steps you can take to protect your online security.

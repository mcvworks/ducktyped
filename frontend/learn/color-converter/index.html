Color Models Explained

Understand how digital color models work -- from the RGB system your screen uses to the HEX shorthand in your stylesheets and the HSL model that makes color manipulation intuitive -- along with alpha transparency, CSS color functions, and accessibility guidelines.

What Are Color Models?

A color model is a mathematical system for describing colors as numbers. Every color you see on a screen is ultimately a combination of numerical values that tell the display hardware how much light to emit at each pixel. Different color models represent the same colors using different coordinate systems, each with its own strengths.

In web development, three color models dominate: RGB (used internally by displays), HEX (a compact notation for RGB values), and HSL (a more intuitive model for humans to reason about). Understanding all three -- and how to convert between them -- is essential for front-end development, design systems, and accessibility work.

The RGB Model

RGB stands for Red, Green, Blue -- the three primary colors of light. It is an additive color model, meaning that colors are created by adding light together. When all three channels are at maximum intensity, the result is white. When all three are at zero, the result is black.

Each channel is represented as an integer from 0 to 255 (8 bits per channel), giving each channel 256 possible values. The total number of representable colors is 256 x 256 x 256 = 16,777,216 -- commonly cited as "16.7 million colors."

rgb(255, 0, 0)      // Pure red
rgb(0, 255, 0)      // Pure green
rgb(0, 0, 255)      // Pure blue
rgb(255, 255, 255)  // White
rgb(0, 0, 0)        // Black
rgb(128, 128, 128)  // Medium gray
rgb(255, 165, 0)    // Orange

The RGB model maps directly to how screens work -- each pixel on an LCD or OLED display contains red, green, and blue sub-pixels whose brightness is controlled independently. This direct correspondence makes RGB the fundamental color model in digital systems.

HEX Notation

HEX color codes are simply RGB values written in hexadecimal (base-16) notation, prefixed with a # sign. Each pair of hex digits represents one channel:

#RRGGBB

#FF0000  = rgb(255, 0, 0)    // Red
#00FF00  = rgb(0, 255, 0)    // Green
#0000FF  = rgb(0, 0, 255)    // Blue
#FFFFFF  = rgb(255, 255, 255) // White
#000000  = rgb(0, 0, 0)      // Black
#FFA500  = rgb(255, 165, 0)  // Orange

Each hex digit represents a value from 0 to 15 (0--9 and A--F). Two hex digits together represent a value from 0 to 255 (00 to FF). The conversion is straightforward: the hex value FF is 15 x 16 + 15 = 255, and A5 is 10 x 16 + 5 = 165.

Shorthand HEX

When each pair of hex digits consists of the same character repeated, CSS allows a three-character shorthand:

#RGB expands to #RRGGBB

#F00 = #FF0000  // Red
#0F0 = #00FF00  // Green
#FFF = #FFFFFF  // White
#333 = #333333  // Dark gray

This shorthand only works when each channel can be expressed as a repeated digit. #FFA500 (orange) has no shorthand equivalent.

Eight-Digit HEX (with Alpha)

Modern CSS supports eight-digit hex codes that include an alpha (transparency) channel as the final two digits:

#RRGGBBAA

#FF000080  // Red at 50% opacity (80 hex = 128 decimal = ~50%)
#00000000  // Fully transparent black
#FFFFFFFF  // Fully opaque white

The alpha value follows the same 00--FF range, where 00 is fully transparent and FF is fully opaque. A four-digit shorthand #RGBA is also supported.

The HSL Model

HSL stands for Hue, Saturation, Lightness. Unlike RGB, which describes how much of each primary light to mix, HSL describes color in terms that are more intuitive for humans to reason about:

  • Hue (0--360) -- The position on the color wheel, measured in degrees. 0 (and 360) is red, 120 is green, 240 is blue, 60 is yellow, 180 is cyan, 300 is magenta.
  • Saturation (0--100%) -- The intensity or vividness of the color. 100% is fully saturated (pure color), 0% is completely desaturated (gray).
  • Lightness (0--100%) -- How light or dark the color is. 0% is black, 100% is white, 50% is the "pure" color at normal brightness.
hsl(0, 100%, 50%)    // Pure red
hsl(120, 100%, 50%)  // Pure green
hsl(240, 100%, 50%)  // Pure blue
hsl(0, 0%, 100%)     // White
hsl(0, 0%, 0%)       // Black
hsl(0, 0%, 50%)      // Medium gray

HSL is especially useful when you need to create color variations programmatically. To lighten a color, increase the lightness. To desaturate it, decrease the saturation. To shift the hue, rotate the hue value. These operations are straightforward in HSL but awkward in RGB, where achieving a "lighter blue" requires knowing how to adjust all three channels proportionally.

HSV/HSB

HSV (Hue, Saturation, Value) -- also called HSB (Hue, Saturation, Brightness) -- is closely related to HSL but defines the third component differently. In HSV, the "value" represents the maximum brightness of the color, while in HSL, "lightness" represents the midpoint between black and white.

HSV is commonly used in color pickers (like those in Photoshop and Figma) because it maps naturally to how users select colors: pick a hue on the wheel, then choose saturation and brightness on a square or triangle. CSS does not natively support HSV notation, so colors selected in HSV-based tools must be converted to RGB, HEX, or HSL for use in stylesheets.

Converting Between Color Models

Since HEX is just a different notation for RGB, converting between them is a matter of changing the number base. The more interesting conversions are between RGB and HSL.

RGB to HEX

Convert each RGB channel value (0--255) to its two-digit hexadecimal representation and concatenate with a # prefix:

rgb(255, 165, 0)
  255 = FF, 165 = A5, 0 = 00
  Result: #FFA500

RGB to HSL (Conceptual)

The conversion from RGB to HSL involves normalizing the RGB values to the 0--1 range, finding the minimum and maximum channel values, and computing:

  • Lightness = (max + min) / 2
  • Saturation = (max - min) / (1 - |2L - 1|), or 0 if max equals min (gray)
  • Hue = determined by which channel is the maximum, using a formula that places the result on the 0--360 degree color wheel

The reverse conversion (HSL to RGB) works through an intermediate chroma value and temporary channel calculations. These formulas are well-documented and implemented in every color library, but the math is precise enough that hand-calculating is impractical -- which is exactly why converter tools exist.

Alpha and Transparency

The alpha channel controls opacity -- how see-through a color is. An alpha of 1 (or 100%) means fully opaque; an alpha of 0 means fully transparent. Alpha can be added to any color model:

rgba(255, 0, 0, 0.5)   // Red at 50% opacity
hsla(0, 100%, 50%, 0.5) // Same color in HSLA
#FF000080               // Same in 8-digit hex

In modern CSS (Level 4), the rgba() and hsla() functions have been merged into rgb() and hsl() -- you can now write rgb(255 0 0 / 0.5) using the space-separated syntax with a slash before the alpha value.

CSS Color Functions

CSS has evolved significantly in its color capabilities. Beyond the classic rgb() and hsl(), modern CSS introduces several powerful color functions:

  • rgb() and hsl() -- The classic functions, now supporting the space-separated syntax: rgb(255 0 0 / 0.5).
  • oklch() -- Uses the OKLCH color space, which provides perceptually uniform lightness and chroma. Colors that have the same lightness value in OKLCH actually appear equally bright to the human eye, which is not true in HSL. This is increasingly recommended for design systems.
  • color-mix() -- Mixes two colors together in a specified color space: color-mix(in srgb, red 50%, blue) produces a purple. This function is powerful for creating dynamic color palettes without JavaScript.
  • Named colors -- CSS defines 148 named colors, from aliceblue to yellowgreen. These are convenient for prototyping but their names are arbitrary and do not form a systematic palette. transparent is a special named color equivalent to rgba(0, 0, 0, 0).

Accessibility and Contrast

Color choice has direct implications for accessibility. The Web Content Accessibility Guidelines (WCAG) define minimum contrast ratios between foreground text and its background:

  • WCAG AA (minimum): contrast ratio of at least 4.5:1 for normal text and 3:1 for large text (18pt or 14pt bold).
  • WCAG AAA (enhanced): contrast ratio of at least 7:1 for normal text and 4.5:1 for large text.

The contrast ratio is calculated from the relative luminance of the two colors, which is derived from their linearized RGB values (not their raw RGB values). A ratio of 1:1 means no contrast (same color), while 21:1 is the maximum (pure black on pure white).

When designing color schemes, always verify contrast ratios for text elements. Beautiful colors that fail contrast requirements make your content inaccessible to users with low vision or color vision deficiency. Tools that calculate contrast ratios can save you from guessing.

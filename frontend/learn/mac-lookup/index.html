MAC Address Lookup Explained

Understand how MAC addresses work, what the OUI reveals about device manufacturers, and how to use duckTyped's MAC lookup tool to identify vendors and troubleshoot network devices.

What Is a MAC Address?

A MAC (Media Access Control) address is a unique hardware identifier assigned to every network interface controller (NIC) -- whether it is an Ethernet port, a Wi-Fi adapter, or a Bluetooth radio. MAC addresses operate at Layer 2 (Data Link Layer) of the OSI model and are used to deliver frames between devices on the same local network segment.

Unlike IP addresses, which are logical and can change depending on the network a device connects to, MAC addresses are typically burned into the hardware by the manufacturer. They provide a persistent, globally unique identity for each network interface. Every Ethernet frame contains both a source and destination MAC address, enabling switches to forward traffic to the correct physical port.

MAC Address Structure

A standard MAC address (also called an EUI-48 address) is 48 bits (6 bytes) long. It is divided into two equal halves:

  • First 3 bytes (24 bits) -- OUI (Organizationally Unique Identifier): This portion identifies the manufacturer or vendor that produced the network interface. The OUI is assigned by the IEEE (Institute of Electrical and Electronics Engineers) to organizations that register for one.
  • Last 3 bytes (24 bits) -- NIC Specific: This portion is assigned by the manufacturer and uniquely identifies the individual network interface within that vendor's product line. The manufacturer is responsible for ensuring no two devices share the same full MAC address.

For example, in the MAC address 00:1A:2B:3C:4D:5E, the OUI is 00:1A:2B (which might identify a specific manufacturer) and the NIC-specific portion is 3C:4D:5E.

OUI -- Organizationally Unique Identifier

The OUI is the cornerstone of MAC address vendor identification. The IEEE maintains a public registry of OUI assignments, which maps each 3-byte prefix to the organization that owns it. Major manufacturers own multiple OUIs -- Apple, for example, has dozens of registered OUI prefixes to accommodate its vast product lines.

There are three sizes of identifier blocks that organizations can register:

  • MA-L (MAC Address Block Large) -- the traditional 24-bit OUI, providing approximately 16.7 million unique addresses per block.
  • MA-M (MAC Address Block Medium) -- a 28-bit prefix, providing approximately 1 million addresses.
  • MA-S (MAC Address Block Small) -- a 36-bit prefix, providing 4,096 addresses. Also known as OUI-36.

By looking up the OUI in the IEEE registry, you can determine which company manufactured a network device -- even without physical access to it. This is the core function of a MAC address lookup tool.

MAC Address Formats

MAC addresses can be written in several different notations depending on the operating system or network equipment in use:

  • Colon notation: 00:1A:2B:3C:4D:5E -- the most common format, used by Linux, macOS, and most networking documentation.
  • Hyphen notation: 00-1A-2B-3C-4D-5E -- used by Windows (ipconfig /all) and the IEEE's official format in registration documents.
  • Dot notation: 001A.2B3C.4D5E -- used by Cisco IOS and other network equipment, grouping bytes into three sets of four hex digits.

All three formats represent the same address. A robust MAC lookup tool accepts any of these formats and normalizes them for lookup.

Special Bits in MAC Addresses

The least significant two bits of the first byte carry special meaning:

Unicast vs. Multicast Bit

The least significant bit (bit 0) of the first octet indicates whether the address is unicast or multicast. If this bit is 0, the address is a unicast address -- it identifies a single network interface. If the bit is 1, the address is a multicast address -- frames sent to this address are received by multiple interfaces that have subscribed to the multicast group. The broadcast address FF:FF:FF:FF:FF:FF is a special case of multicast where all devices on the segment receive the frame.

Locally Administered Bit

The second least significant bit (bit 1) of the first octet indicates whether the address is universally administered (assigned by the manufacturer via the OUI system) or locally administered (overridden by a network administrator or software). When this bit is 0, the address is universally administered. When it is 1, the address has been set locally and may not correspond to any OUI in the IEEE registry.

Locally administered addresses are commonly encountered in virtualized environments (hypervisors often generate MAC addresses with the locally administered bit set) and in modern privacy features on mobile devices.

Use Cases for MAC Address Lookup

MAC address lookup serves several practical purposes in network management and security:

  • Identifying device manufacturers -- when you see an unknown device on your network, looking up its MAC address reveals the manufacturer. This can help you determine if it is a laptop (Dell, Lenovo), a phone (Apple, Samsung), a smart home device (Amazon, Google), or potentially an unauthorized device.
  • Network inventory and asset management -- organizations use MAC address data to maintain inventories of network-connected devices, track hardware across locations, and verify that only authorized equipment is connected.
  • Troubleshooting connectivity issues -- when diagnosing network problems, knowing the vendor behind a MAC address helps identify the type of device and narrow down potential driver, firmware, or configuration issues.
  • Security monitoring -- intrusion detection systems and network access control (NAC) solutions use MAC addresses to enforce policies about which devices are permitted on the network.

MAC Spoofing

MAC spoofing is the practice of changing a device's MAC address to impersonate another device or bypass MAC-based access controls. While MAC addresses are burned into hardware, most operating systems allow software-level overrides. Attackers may spoof MAC addresses to evade network access control lists, bypass captive portals, or impersonate trusted devices.

Because of the ease of spoofing, MAC addresses should never be used as the sole mechanism for network authentication or security. Modern networks use 802.1X authentication, certificates, or other cryptographic methods in addition to MAC-based controls.

Randomized MAC Addresses -- Privacy on Mobile Devices

Starting with iOS 14 and Android 10, mobile operating systems began using randomized (private) MAC addresses when scanning for and connecting to Wi-Fi networks. This is a privacy feature designed to prevent tracking of users across different locations based on their device's MAC address.

When MAC randomization is active, the device generates a different MAC address for each network it connects to, and the locally administered bit is set to 1. This means the OUI will not correspond to Apple, Samsung, or any real manufacturer -- instead, it will appear as a locally administered address.

This has significant implications for network administrators:

  • MAC-based device identification becomes unreliable for mobile clients.
  • DHCP reservations tied to MAC addresses may break when the randomized address changes.
  • Network analytics that count unique devices by MAC address will overcount mobile users.

Most enterprise network solutions have adapted by using alternative identification methods such as 802.1X certificates or device management profiles.

How to Find Your MAC Address

You can find the MAC address of your own device using the following commands:

  • Windows: Open Command Prompt and run ipconfig /all. Look for "Physical Address" under your active network adapter.
  • macOS: Open Terminal and run ifconfig en0 (for Wi-Fi) or ifconfig en1. The MAC address is listed next to "ether".
  • Linux: Open a terminal and run ip link show or ifconfig. The MAC address appears next to "link/ether" or "HWaddr".
  • iOS: Go to Settings > General > About and look for "Wi-Fi Address".
  • Android: Go to Settings > About Phone > Status (or Wi-Fi MAC Address, depending on the manufacturer).

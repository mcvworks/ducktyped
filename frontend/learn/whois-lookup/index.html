WHOIS Lookup Explained

Understand how the WHOIS protocol works, what domain registration data is available, how privacy regulations have changed the landscape, and how to use duckTyped's WHOIS lookup tool to investigate domain ownership and registration details.

What Is WHOIS?

WHOIS is a query-and-response protocol used to look up information about registered domain names, IP address blocks, and autonomous systems. When you perform a WHOIS lookup on a domain like example.com, the protocol returns a structured set of data about that domain's registration -- who registered it, when it was created, when it expires, and which nameservers it uses.

Think of WHOIS as a public registry for the internet. Just as property records let you find out who owns a piece of land, WHOIS records let you find out who is responsible for a domain name or IP address block. This transparency has been a foundational principle of internet governance since the early days of the network.

History of the WHOIS Protocol

The WHOIS protocol dates back to the early 1980s, making it one of the oldest protocols still in active use on the internet. It was originally specified in RFC 812 (1982) by Ken Harrenstien and Vic White at the Network Information Center (NIC) at Stanford Research Institute. At that time, the internet was a small research network, and WHOIS served as a simple directory of network users and hosts.

As the internet grew, WHOIS evolved to track domain name registrations. When the domain name system was formalized in 1985, WHOIS became the standard mechanism for querying registration data. The protocol operates on TCP port 43 -- a client sends a text query, and the server responds with a plain-text answer. This simplicity has been both a strength and a limitation, as the unstructured text format makes automated parsing unreliable across different registrars.

Over the decades, WHOIS data became distributed across hundreds of registrars and registries, each with slightly different formatting. ICANN (the Internet Corporation for Assigned Names and Numbers) has worked to standardize the output, but inconsistencies remain. This fragmentation was one of the motivations behind the development of RDAP, a modern replacement protocol.

WHOIS Data Fields

A typical WHOIS response contains several categories of information about a domain registration. Understanding these fields is essential for interpreting lookup results.

Registrar Information

The registrar is the company through which the domain was registered (e.g., GoDaddy, Namecheap, Cloudflare Registrar, Google Domains). WHOIS records include the registrar's name, IANA ID, abuse contact email, and referral URL. This information helps you identify who manages the domain's registration and where to report abuse.

Registrant, Admin, and Tech Contacts

WHOIS records traditionally include three contact types: the registrant (the domain owner), the administrative contact (responsible for administrative decisions), and the technical contact (responsible for technical operations). Each contact entry may include a name, organization, street address, city, country, phone number, and email address. However, as discussed below, privacy regulations have significantly reduced the visibility of this data.

Important Dates

Every WHOIS record includes critical dates:

  • Creation Date -- when the domain was first registered.
  • Updated Date -- when the registration record was last modified (e.g., after a DNS change or registrar transfer).
  • Expiration Date -- when the domain registration expires. If not renewed before this date, the domain may become available for others to register.

These dates are particularly useful for assessing domain age (older domains are generally more trustworthy), monitoring renewal deadlines, and investigating potentially suspicious newly registered domains.

Nameservers

The WHOIS record lists the authoritative nameservers for the domain. These are the DNS servers that hold the domain's DNS records (A, MX, TXT, etc.). For example, a domain using Cloudflare might show nameservers like ada.ns.cloudflare.com and bob.ns.cloudflare.com. This information is useful for confirming that DNS delegation is configured correctly.

Domain Status Codes

WHOIS records include EPP status codes that indicate the current state of the domain. Common codes include:

  • clientTransferProhibited -- the registrar has locked the domain to prevent unauthorized transfers.
  • serverDeleteProhibited -- the registry has locked the domain to prevent deletion.
  • redemptionPeriod -- the domain has expired and is in a grace period before being released.
  • pendingDelete -- the domain is scheduled for deletion and will soon be available for registration.

RDAP vs WHOIS

The Registration Data Access Protocol (RDAP) is the modern successor to WHOIS, defined in a series of RFCs published in 2015 (RFC 7480--7485). RDAP addresses many of WHOIS's long-standing limitations:

  • Structured data -- RDAP returns responses in JSON format, making automated parsing reliable and consistent across providers. WHOIS returns unstructured plain text that varies by registrar.
  • Standardized queries -- RDAP uses RESTful HTTP/HTTPS endpoints with well-defined URL patterns, while WHOIS relies on a simple TCP text protocol with no standardized query format.
  • Access control -- RDAP supports authentication and differentiated access levels, allowing registrars to show more data to authorized users (such as law enforcement) while restricting public access. WHOIS has no built-in access control.
  • Internationalization -- RDAP natively supports Unicode and internationalized domain names (IDNs), whereas WHOIS has limited support for non-ASCII characters.
  • Secure transport -- RDAP runs over HTTPS by default, encrypting queries and responses. WHOIS transmits data in plain text over TCP.

ICANN has mandated that all gTLD registries and registrars support RDAP. While WHOIS remains widely used, RDAP is increasingly becoming the primary lookup mechanism for domain registration data.

Privacy and GDPR Redaction

The introduction of the General Data Protection Regulation (GDPR) in May 2018 fundamentally changed the WHOIS landscape. Because WHOIS records traditionally contained personal information -- names, addresses, phone numbers, and email addresses of domain registrants -- European privacy law required that this data be protected.

In response, ICANN adopted a Temporary Specification for gTLD Registration Data that allowed registrars to redact personal information from public WHOIS output. Today, most WHOIS lookups for gTLDs return redacted contact information, with fields replaced by labels like REDACTED FOR PRIVACY or generic registrar proxy addresses.

Some registrants had already used WHOIS privacy services (also called domain privacy or proxy registration) before GDPR. These services replace the registrant's personal information with the privacy provider's details in the public WHOIS record. Post-GDPR, this level of redaction has become the default for most registrars, even without an explicit privacy service.

Legitimate parties such as law enforcement, intellectual property holders, and cybersecurity researchers can still request access to full registration data through registrar-specific processes, but casual public lookups now return limited information for most domains.

Common Use Cases

Despite privacy redaction, WHOIS lookups remain valuable for many practical purposes.

Checking Domain Availability

Before registering a new domain, a WHOIS lookup confirms whether the domain is already taken. If the query returns no results or an NXDOMAIN-equivalent status, the domain may be available for registration. This is often the first step in the domain purchasing process.

Investigating Domain Ownership

Security professionals, journalists, and researchers use WHOIS data to investigate who is behind a particular domain. Even with GDPR redaction, useful information remains -- the registrar name, creation date, and nameservers can reveal patterns. For example, a phishing domain registered yesterday through a budget registrar raises different flags than a 15-year-old domain registered through a reputable provider.

Expiry Monitoring

Organizations monitor the expiration dates of their own domains and critical third-party domains to prevent accidental lapses. An expired domain can be registered by a malicious actor who then impersonates the original owner -- a technique known as domain hijacking. Automated WHOIS monitoring tools alert administrators before expiration dates arrive.

Brand Protection

Companies use WHOIS lookups to identify domains that infringe on their trademarks -- for example, typosquatting domains like gooogle.com or amaz0n.com. The registration data, particularly the registrar and creation date, helps legal teams build cases for domain dispute resolution through ICANN's Uniform Domain-Name Dispute-Resolution Policy (UDRP).

Security Analysis

When investigating suspicious emails, links, or network traffic, WHOIS data helps analysts determine whether a domain is legitimate. Newly registered domains, domains with privacy-protected registration, and domains using certain budget registrars are common indicators of malicious activity -- though none of these factors alone is conclusive.

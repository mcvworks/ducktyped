Port Scanning Explained

Understand how network ports work, what services run on common port numbers, how port scanning techniques operate, and how to use duckTyped's port scanner to check which ports are open on any host.

What Are Network Ports?

A network port is a logical endpoint for communication on a computer. While an IP address identifies a specific machine on a network, a port number identifies a specific service or application running on that machine. Together, an IP address and port number form a socket -- the complete address needed to establish a network connection.

Port numbers are 16-bit unsigned integers, ranging from 0 to 65,535. When your browser connects to a web server, it connects to the server's IP address on port 80 (HTTP) or port 443 (HTTPS). Meanwhile, your browser uses a randomly assigned high-numbered port on your own machine for its side of the connection. This pairing allows a single server to handle thousands of simultaneous connections to different services.

TCP vs UDP

Ports operate within two primary transport-layer protocols, each suited to different types of communication.

TCP (Transmission Control Protocol) is connection-oriented. Before any data is exchanged, the client and server perform a three-way handshake: the client sends a SYN (synchronize) packet, the server responds with a SYN-ACK (synchronize-acknowledge), and the client sends a final ACK (acknowledge). This process establishes a reliable, ordered, and error-checked connection. TCP is used for web traffic, email, file transfers, and any application where data integrity matters.

UDP (User Datagram Protocol) is connectionless. There is no handshake -- the sender simply transmits packets (datagrams) to the destination without waiting for confirmation. This makes UDP faster and more lightweight than TCP, but it provides no guarantees about delivery, ordering, or integrity. UDP is used for DNS queries, video streaming, online gaming, and VoIP, where speed is more important than perfect reliability.

Port Number Ranges

The Internet Assigned Numbers Authority (IANA) divides the port number space into three ranges:

Well-Known Ports (0--1023)

These ports are reserved for standard system services and protocols. On Unix-like operating systems, binding to a well-known port typically requires root or administrator privileges. Examples include port 22 (SSH), port 80 (HTTP), and port 443 (HTTPS). These assignments are globally recognized, so any administrator or security professional immediately knows what service to expect on these ports.

Registered Ports (1024--49151)

These ports are assigned by IANA to specific applications and services upon request, but they do not require special privileges to use. Examples include port 3306 (MySQL), port 5432 (PostgreSQL), and port 8080 (commonly used as an alternative HTTP port). While these assignments are standardized, nothing prevents a non-standard application from using a registered port number.

Ephemeral Ports (49152--65535)

Also called dynamic or private ports, these are used by the operating system for temporary, outgoing connections. When your browser connects to a web server, the OS assigns an ephemeral port for the client side of the connection. These ports are allocated dynamically and released when the connection closes.

Common Ports and Services

Knowing which services typically run on which ports is fundamental to network administration and security analysis. Here are the ports you will encounter most frequently:

  • Port 21 -- FTP (File Transfer Protocol). Used for file transfers, though increasingly replaced by SFTP and SCP due to security concerns.
  • Port 22 -- SSH (Secure Shell). Provides encrypted remote login and command execution. Also used for SFTP and SCP file transfers.
  • Port 25 -- SMTP (Simple Mail Transfer Protocol). Used for sending and relaying email between mail servers.
  • Port 53 -- DNS (Domain Name System). Used for domain name resolution queries and zone transfers. Operates on both TCP and UDP.
  • Port 80 -- HTTP (Hypertext Transfer Protocol). The standard port for unencrypted web traffic.
  • Port 110 -- POP3 (Post Office Protocol v3). Used by email clients to retrieve mail from a server.
  • Port 143 -- IMAP (Internet Message Access Protocol). A more advanced email retrieval protocol that keeps mail on the server.
  • Port 443 -- HTTPS (HTTP Secure). The standard port for encrypted web traffic using TLS/SSL.
  • Port 993 -- IMAPS. IMAP over TLS/SSL for encrypted email retrieval.
  • Port 995 -- POP3S. POP3 over TLS/SSL for encrypted email retrieval.
  • Port 3306 -- MySQL. The default port for MySQL database connections.
  • Port 3389 -- RDP (Remote Desktop Protocol). Used for remote graphical access to Windows machines.
  • Port 5432 -- PostgreSQL. The default port for PostgreSQL database connections.
  • Port 6379 -- Redis. The default port for the Redis in-memory data store.
  • Port 8080 -- HTTP Alternative. Commonly used for web servers, proxies, and development environments.
  • Port 27017 -- MongoDB. The default port for MongoDB database connections.

How Port Scanning Works

Port scanning is the process of probing a host's ports to determine which ones are open, closed, or filtered. This information reveals which services are running and potentially reachable from the network.

TCP Connect Scan

The simplest scanning technique is the TCP connect scan. The scanner attempts to complete a full three-way handshake with each target port. If the handshake succeeds (the server responds with SYN-ACK), the port is open and a service is listening. If the server responds with a RST (reset) packet, the port is closed. If there is no response or an ICMP unreachable message is returned, the port is likely filtered by a firewall. This scan is reliable but easily detected because it creates complete connections that appear in server logs.

SYN Scan (Half-Open Scan)

A SYN scan sends only the initial SYN packet without completing the handshake. If the server responds with SYN-ACK, the scanner knows the port is open and immediately sends a RST packet to tear down the connection before it fully establishes. This technique is faster than a full connect scan and historically was harder to detect (earning it the name "stealth scan"), though modern intrusion detection systems readily identify SYN scans.

Firewalls and Port Filtering

A firewall is a network security device (hardware or software) that controls incoming and outgoing traffic based on predefined rules. Firewalls are the primary reason ports appear as "filtered" during a scan.

Firewalls can implement different strategies for handling unwanted traffic:

  • Drop -- the firewall silently discards the packet with no response. The scanner sees a timeout and reports the port as filtered. This is the most common approach, as it reveals the least information to potential attackers.
  • Reject -- the firewall sends back a RST packet or ICMP unreachable message. The scanner reports the port as closed, even though the firewall intervened rather than the actual service.

Best practice is to configure firewalls to allow only the ports required for legitimate services and drop all other traffic. For example, a web server should typically expose only ports 80 and 443, with all other ports blocked.

Security Implications

Port scanning is a double-edged tool. System administrators use it to audit their own infrastructure -- verifying that only intended services are exposed, confirming firewall rules are working correctly, and identifying unauthorized services. Regular port scanning is a fundamental part of security hygiene.

However, port scanning is also the first step in many attack methodologies. An attacker scans a target to discover open ports, identifies the services behind them, and then looks for known vulnerabilities in those services. For this reason, security teams monitor for unexpected port scans as a potential indicator of reconnaissance activity.

Key security practices related to ports include:

  • Minimize your attack surface -- only open the ports that are absolutely necessary for your services.
  • Keep services updated -- an open port running outdated software with known vulnerabilities is a prime target.
  • Use non-standard ports with caution -- running SSH on port 2222 instead of 22 may reduce automated scanning noise but is not a substitute for proper security.
  • Monitor for changes -- if a new port suddenly appears open on your server, investigate immediately.

URL Metadata Explained

Understand what URL metadata is, how Open Graph and Twitter Card tags control social sharing previews, which SEO meta tags matter most, and how to inspect and optimize metadata for any webpage using duckTyped's URL Metadata Extractor.

What Is URL Metadata?

URL metadata refers to the structured information embedded in a webpage's HTML <head> section that describes the page's content to browsers, search engines, social platforms, and other automated systems. Unlike the visible content on a page, metadata is invisible to casual visitors but plays a critical role in how the page appears in search results, social media feeds, messaging apps, and browser tabs.

When you share a link on Twitter, LinkedIn, Slack, or iMessage, the platform fetches the target URL and reads its metadata to generate a rich preview -- the card with a title, description, and image that appears alongside the link. Without proper metadata, shared links appear as plain text URLs, which receive dramatically fewer clicks and engagement.

Standard SEO Meta Tags

Before social-specific protocols emerged, a handful of standard HTML meta tags defined how pages were described to search engines and browsers. These remain foundational.

Title Tag

The <title> element is arguably the single most important piece of metadata on any page. It appears in browser tabs, search engine results pages (SERPs), and as the default text when bookmarking a page. Best practices recommend keeping titles under 60 characters to avoid truncation in search results, placing the primary keyword near the beginning, and making each page's title unique across your site. The title tag is not a meta tag technically -- it is its own HTML element -- but it is universally considered part of a page's metadata.

Meta Description

The <meta name="description"> tag provides a brief summary of the page's content. Search engines often display this as the snippet beneath the title in results. While Google has confirmed that meta descriptions are not a direct ranking factor, a well-written description significantly improves click-through rate (CTR), which indirectly affects rankings. Keep descriptions between 120 and 160 characters, include relevant keywords naturally, and write compelling copy that encourages clicks.

Canonical Tag

The <link rel="canonical"> tag tells search engines which URL is the authoritative version of a page. This is essential for preventing duplicate content issues when the same content is accessible at multiple URLs (e.g., with and without trailing slashes, with query parameters, or across HTTP and HTTPS). The canonical tag consolidates ranking signals to a single URL.

Robots Meta Tag

The <meta name="robots"> tag controls how search engines crawl and index the page. Common directives include index/noindex (whether to include the page in search results) and follow/nofollow (whether to follow links on the page). You can also target specific bots with tags like <meta name="googlebot">. This is distinct from the robots.txt file, which controls crawl access at the path level.

Open Graph Protocol

The Open Graph protocol was introduced by Facebook in 2010 to give website owners control over how their pages appear when shared on social media. It uses meta tags with the property attribute and the og: prefix. Open Graph has become the de facto standard for social sharing metadata, supported by Facebook, LinkedIn, Pinterest, Discord, Slack, iMessage, and many other platforms.

Essential Open Graph Tags

  • og:title -- the title of the page as it should appear in social previews. This can differ from the HTML <title> tag, allowing you to optimize separately for search engines and social platforms.
  • og:description -- a brief description of the content, typically displayed below the title in social cards. Keep it under 200 characters for optimal display.
  • og:image -- the URL of the image displayed in the social preview. This is the single most impactful Open Graph tag, as posts with images receive significantly more engagement. Use an absolute URL and ensure the image is at least 1200 x 630 pixels for high-quality display across platforms.
  • og:url -- the canonical URL of the page. Social platforms use this to consolidate engagement metrics (likes, shares) across different URL variations.
  • og:type -- the type of content. Common values include website, article, product, and profile. The type determines which additional properties are available (e.g., article:published_time).
  • og:site_name -- the name of the overall website, displayed alongside the page title to provide context.

Additional Open Graph Tags

Beyond the essentials, Open Graph supports tags for og:locale (language and territory), og:video (video content URL), og:audio (audio content URL), and og:image:width / og:image:height (which help platforms render the image correctly without downloading it first). The article type adds article:published_time, article:modified_time, article:author, and article:tag.

Twitter Card Tags

Twitter Cards are Twitter's (now X's) proprietary metadata system for rich link previews. While Twitter does fall back to Open Graph tags when its own tags are missing, dedicated Twitter Card tags give you more precise control over how your content appears on the platform.

Card Types

  • summary -- a compact card with a small square image, title, and description. Suitable for general content.
  • summary_large_image -- a card with a large, prominent image above the title and description. Ideal for articles, blog posts, and visual content.
  • player -- embeds a media player for video or audio content directly in the timeline.
  • app -- displays mobile app information with a direct download link.

Key Twitter Tags

  • twitter:card -- specifies the card type (required). If missing, Twitter will not generate a rich preview.
  • twitter:title -- the title for the card. Falls back to og:title if not specified.
  • twitter:description -- the description text. Falls back to og:description.
  • twitter:image -- the image URL. For summary_large_image, use images with a 2:1 aspect ratio, minimum 300 x 157 pixels, maximum 4096 x 4096 pixels.
  • twitter:site -- the Twitter handle of the website (e.g., @ducktyped).
  • twitter:creator -- the Twitter handle of the content author.

Structured Data and JSON-LD

Structured data provides machine-readable information about a page's content using standardized vocabularies, primarily Schema.org. While not visible on the page, structured data helps search engines understand the content more deeply and can trigger rich results in SERPs -- enhanced search listings with star ratings, images, FAQ accordions, event dates, recipe details, and more.

The recommended format is JSON-LD (JavaScript Object Notation for Linked Data), embedded in a <script type="application/ld+json"> tag in the page's head. Google explicitly prefers JSON-LD over older formats like Microdata and RDFa. Common schema types include Article, Product, FAQPage, Organization, BreadcrumbList, and HowTo.

Favicons and Touch Icons

Favicons are the small icons displayed in browser tabs, bookmarks, and history. Modern browsers support multiple icon formats and sizes:

  • <link rel="icon"> -- the standard favicon, typically in ICO, PNG, or SVG format.
  • <link rel="apple-touch-icon"> -- used when users add your site to their iOS home screen. Apple recommends a 180 x 180 pixel PNG.
  • <link rel="manifest"> -- points to a web app manifest file (JSON) that defines icons for Progressive Web Apps (PWAs), including various sizes for Android devices.

Common Metadata Mistakes

Even experienced developers make metadata mistakes that hurt their site's appearance in search results and social feeds:

  • Missing og:image -- the most impactful error. Without a share image, social previews are text-only and receive far less engagement. Always include an og:image tag with a high-quality image.
  • Wrong image dimensions -- images that are too small, too large, or have the wrong aspect ratio get cropped awkwardly or rejected entirely. Use 1200 x 630 pixels for Open Graph images.
  • Duplicate titles and descriptions -- using the same title and meta description across multiple pages confuses search engines and makes your search results indistinguishable. Every page should have unique metadata.
  • Relative image URLs -- Open Graph and Twitter Card tags require absolute URLs (starting with https://). Relative paths like /images/share.png will not work.
  • Missing canonical tag -- without a canonical tag, search engines must guess which version of your page is authoritative, potentially splitting ranking signals across duplicates.
  • Overly long titles -- titles exceeding 60 characters get truncated in search results, potentially cutting off important keywords or your brand name.
  • Stale cached previews -- social platforms cache metadata aggressively. After updating your tags, use each platform's cache invalidation tool (e.g., Facebook's Sharing Debugger, Twitter's Card Validator) to refresh the preview.
